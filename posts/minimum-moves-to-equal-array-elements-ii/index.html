Minimum Moves to Equal Array Elements II

June 30, 2022

Question:#

Given an integer array nums of size n, return the minimum number of moves required to make all array elements equal.

In one move, you can increment or decrement an element of the array by 1.

Test cases are designed so that the answer will fit in a 32-bit integer.

Example 1:

Input: nums = [1,2,3] Output: 2 Explanation: Only two moves are needed (remember each move increments or decrements one element): [1,2,3] => [2,2,3] => [2,2,2]

Example 2:

Input: nums = [1,10,2,9] Output: 16

Constraints:

  • n == nums.length
  • 1 <= nums.length <= 105
  • -109 <= nums[i] <= 109

Explanation:#

So we need to find the number (say x) that has the smallest distance between it and all of the other elements. Once you have this number you can map(lambda a: abd(x - a), nums) to get the number of moves required for each number to make it into x. We can sum this list to get our total.

The niave approach is going to be to check the distance between each number in range and all of the other numbers and comparing their average distances. This is super slow because the max number is 10^9. Even if we assume x is a number in the list this would be O(n^2) and the list is size 10^5.

The answer is that x = median. This seems too simple. However, remember if you have an x and you add 1 to it all of the values larger than it in the list have to move less to get to it where as all of the numbers smaller than it have to move more to get to it. So each number to the left of x in your ordered list will require +1 moves to reach x+1 and to the right will require -1 moves. Thus, if you have an have the median (the middle of the sorted list) you have the same number of values to the left as to the right. Clearly, median + 1 is going to have a larger number of values to the left of it than to the right and the number of moves to be larger as each number to the left has to make an extra move. Having the same number of values to the left and right of the number subsequently will always be the correct x.

Python Solution:

class Solution:
    def minMoves2(self, nums: List[int]) -> int:
        median = sorted(nums)[len(nums)//2]
        return sum(map(lambda a: abs(a - median), nums))
