Binary Tree Right Side View

2022-07-11

#algorithms#code

Link to problem

Problem Explanation

This question is asking you to return a list that contains the value of nodes collected by taking the right most element of each level. So [1,2,3] should return [1,3] and [1,2] should return [1,2]. This means we can’t simply append the value and then set the node to its right child, because if we hit a null value, we need to append the next most right value on that level. I’d recommend you complete the [[Fibbonacci Number]] question before you attempt this question, as trees are inherently a recursive data type, so it’s important you have a strong understanding of recursion before continuing.

Solution

To begin, we should notice that in the worst case, we need to look at every node. If we hit the end of the tree, we would need to traverse to every leaf to be sure that no element sat on that row. Now because we know that our solution must be at least O(n) we have been freed up to use O(n) operations. Note, any constant number of O(n) operations equates to O(n). This means if we don’t like the data structure we are using, provided we can create a new data structure in O(n) time we can hop between them as we please. For this question, a data structure that this would be very easy with is a list of lists where each sublist represents a level on the tree. If we had this data structure, then we could just return a list with the last element of each sublist. Ok, let’s start coding.

The first thing we want to do is link each node with its respective level. We can make a helper function to do this. Helper functions are really useful for recursive data types like trees because it allows you to add extra arguments that can act as a store of state in your recursive calls. Below, we use the layer argument to keep track of which layer we are looking at. We could also of had this as a variable outside the function that we alter, but this would make the function less pure and more prone to bugs.

    def layered(self, root, layer, layers):
        if root is not None:
            layers[root] = layer
            self.layered(root.left, layer + 1, layers)
            self.layered(root.right, layer + 1, layers)
        return layers

Despite what I said about purity, I elected to alter an outside of function layers dictionary for the sake of not having to return a conglomeration of left and right function calls.

So now that we have the layer of each node we can populate a list of lists where each sublist is another layer of our tree.

    def listOfLists(self, root, layered, lol):
        if root is not None:
            index = layered[root]
            lol[index].append(root.val)
            self.listOfLists(root.left, layered, lol)
            self.listOfLists(root.right, layered, lol)
        return lol

The preconditions for this function are that layered is already a populated dictionary and that lol is a list populated with however many empty lists we have layers.

And now plugging it all together we have the final solution

class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        layers = {}
        lol = []
        out = []
        if root is None:
            return []
        self.layered(root, 0, layers)
        for i in range(max(layers.values()) + 1):
            lol.append([])
        self.listOfLists(root, layers, lol)
        for lo in lol:
            out.append(lo[-1])
        return out      
        
    def layered(self, root, layer, layers):
        if root is not None:
            layers[root] = layer
            self.layered(root.left, layer + 1, layers)
            self.layered(root.right, layer + 1, layers)
        return layers
    
    def listOfLists(self, root, layered, lol):
        if root is not None:
            index = layered[root]
            lol[index].append(root.val)
            self.listOfLists(root.left, layered, lol)
            self.listOfLists(root.right, layered, lol)
        return lol
